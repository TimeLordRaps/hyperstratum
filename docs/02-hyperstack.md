# Hyperstack

The hyperstack is the dependency order of the core ontology.

```txt
Universe / Reality
        ↓
Form / Frame / Base / Meta / Hyper
        ↓
Hypertopology
        ↓
Hypermanifold
        ↓
Hyperfield
        ↓
Hypernode
        ↓
Hyperconnection / Hyperrelation / Hyperemergent
        ↓
Recursive-graph / Self-graph / Mira-graph
        ↓
Hyperstructure
        ↓
Hyperform
        ↓
Hyperrepresentation
```

## Universe and reality

```txt
Universe = whole containing order.
Reality = situated slice of the universe from within a frame, position, or configuration of conditions.
```

The universe is the whole. A reality is a local slice of the whole.

## Hypertopology

```txt
Hypertopology = nearness-law across recursive form-levels.
```

A hypertopology defines nearness, continuity, boundary, closure, and transformation across recursive form-levels.

Two forms may be near in different ways:

```txt
base-near
meta-near
causally near
semantically near
teleologically near
formally near
path-preserving near
```

## Hypermanifold

```txt
Hypermanifold = navigable recursive space.
```

A hypermanifold is a locally navigable space of recursive derivation, where frames become forms, forms become components, and components become part of higher whole-forms.

## Hyperfield

```txt
Hyperfield = distributed potential/substance across recursive form-levels.
```

A hyperfield is a recursively connected distribution of values, meanings, forces, or potentials across a base form, its meta-forms, and the transformation-relations among them.

## Hypernode

```txt
Hypernode = local recursive form-point.
```

A hypernode is a localizable unit in a hyperstructure whose identity includes base appearance, meta-structure, connection-potential, and role inside a larger whole-form.

## Hyperconnection

```txt
Hyperconnection = cross-level connective path.
```

A hyperconnection links forms, nodes, fields, or structures across recursive levels.

## Hyperrelation

```txt
Hyperrelation = rule-governed cross-level relation.
```

A hyperrelation specifies the rule, role, meaning, or transformation governing a connection.

## Hyperemergent

```txt
Hyperemergent = emerging cross-level coherence before stabilization.
```

A hyperemergent is a newly emerging coherence before it stabilizes into a named hyperrelation, graph, or hyperform.

## Graph layer

```txt
Recursive-graph = graph representation of recursive form-structure.
Self-graph = graph whose subject is represented in relation to itself.
Mira-graph = reflective graph from an interpretive position.
```

These can compose into terms such as:

```txt
self-mira-recursive-graph
```

## Hyperstructure

```txt
Hyperstructure = whole recursive object, including all levels and all connections between levels.
```

## Hyperform

```txt
Hyperform = closed or closing identity of a hyperstructure.
```

## Hyperrepresentation

```txt
Hyperrepresentation = recursive representation across base, meta, path, frame, receiver, self-reflection, ambiguity, and whole-form context.
```

Construction order places hyperrepresentation late. Interpretive order lets hyperrepresentation wrap the whole stack.
