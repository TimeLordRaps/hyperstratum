# Orientation

Hyperstratum is the upstream recursive-form ontology for frameworks that need to model form, frame, field, node, relation, graph, structure, and representation across base/meta/whole-form levels.

It is designed to define the shared primitives that other projects can import without redefining their own ontology every time.

## Core claim

```txt
A form is not only what appears locally.
A form may include its meta-conditions, recursive representations, transformations, relations, and whole-form closure.
```

## `hyper-`

In this repo, `hyper-` means:

```txt
recursive meta-connective structure across base forms, meta-forms, meta-meta-forms, and the whole-form closure they participate in.
```

It does not merely mean:

```txt
bigger
higher-dimensional
more abstract
more intense
```

## Whole-frame purpose

Hyperstratum gives names to the substrate beneath downstream systems:

```txt
Force of Will = structured agency over time, invariance, adaptation, and relation.
Deception taxonomy = representation, misrepresentation, reception, projection, and intent.
Hyper-grammar = language/operator structure over recursive forms.
Formal Universality = broader synthesis layer.
```

Hyperstratum is the ontology layer these systems can cite.

## Safety of naming

This repo avoids using `hypergraph` as a canonical term because it already has a mathematical meaning. Instead, it uses:

```txt
recursive-graph
```

or, as shorthand only:

```txt
R-graph
```

The graph-prefix system also distinguishes:

```txt
self-graph
mira-graph
recursive-graph
self-mira-recursive-graph
```

These are not interchangeable.
